## Procédures de restauration

### 1. Restaurer le code (rollback git)

```bash
git checkout backup-2026-04-18-prod-v2

git reset --hard backup-2026-04-18-prod-v2
git push --force origin main  # ATTENTION : destructif
```

### 2. Restaurer les configs DB (templates)

Le dump JSON contient `template.config` complet par slug. Pour rollback un seul
slug :

```js
// node restore-template.mjs
import { readFileSync } from 'node:fs';
const dump = JSON.parse(readFileSync('backups/templates-config-2026-04-18T22-49-40.json', 'utf8'));
const slug = process.argv[2]; // ex: 'patchdouleurtk'
const t = dump.templates[slug];

const login = await fetch('https://gs-pipeline-app-2.vercel.app/api/auth/login', {
  method: 'POST', headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: 'admin@gs-pipeline.com', password: 'admin123' }),
}).then(r => r.json());

await fetch(`https://gs-pipeline-app-2.vercel.app/api/templates/${t.id}`, {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${login.token}` },
  body: JSON.stringify({ config: JSON.stringify(t.config) }),
});
```

### 3. Restaurer le VPS

#### Restaurer .htaccess uniquement (rapide)
```bash

# choisir un backup (ex: .htaccess.bak-pre-https-force-*)
ssh -i ~/.ssh/obrille_temp obrilleshell@vps112526.serveur-vps.net 'cp /web/.htaccess.bak-pre-https-force-XXX /web/.htaccess'
```

#### Restaurer tout (.htaccess + landings-app + api) depuis l'archive
```bash
# Si l'archive distante existe encore :
ssh -i ~/.ssh/obrille_temp obrilleshell@vps112526.serveur-vps.net 'cd /web && tar -xzf /tmp/vps-prod-backup-20260418-224956.tar.gz'

# Sinon, ré-uploader depuis le local (script equivalent à push-vps-file via base64) :
node scripts/pull-vps-file.mjs /tmp/restore-readback.tar.gz /tmp/sanity.tar.gz  # juste lire un fichier
```
